# コーポレートサイト化 作業計画

## 1. 調査タスクのチケット化
- **T1: 商品依存コンポーネント洗い出し**
  - 対象: `src/lib/shopify`、`src/layouts/components`、`src/pages/products`、`src/cartStore.ts`
  - 成果物: 削除候補リストと依存関係図
- **T2: 企業サイトに必要なコンテンツ構成定義**
  - 対象: `src/content` 配下のコレクション構造、`config/*.json`
  - 成果物: 新しいコンテンツタイプ一覧、必要なフィールド定義
- **T3: 外部連携とフォーム要件確認**
  - 対象: 問い合わせフォーム、ニュース/ブログ配信、分析ツール
  - 成果物: 使用サービス決定、必要なAPIキー・設定の確認メモ
- **T4: 既存スタイル／テーマ依存調査**
  - 対象: `src/styles/*.css`、`scripts/themeGenerator.js`
  - 成果物: 商品向けスタイルの識別と再利用可否評価

## 2. 不要機能削除の実装順序
1. **ショップ機能の停止**: `src/lib/shopify` API 呼び出しとカートストアを無効化し、ビルド時エラーが出ないか確認
2. **商品UIコンポーネント削除**: Product系（`ProductGrid.tsx`、`ProductList.tsx` など）とカート系（`NavUser.tsx`、`ProductFilters.tsx` 等）を段階的に削除
3. **商品ページのルーティング整理**: `src/pages/products` ディレクトリを廃止し、404やリダイレクト方針を決定
4. **商品データ資産の削除**: `public/products.csv`、商品画像、`scripts` 内の商品用スクリプトを整理

## 3. 新構成導入の実装順序
1. **コンテンツモデル再構築**: `src/content` をサービス・実績・チーム・ニュースなどの企業項目にリネーム／追加
2. **ナビゲーション・設定更新**: `config/menu.json`、`config/theme.json` を新構成に合わせて改修
3. **レイアウト再設計**: `Base.astro`、`Header.astro`、`Footer.astro` を企業サイトの主要導線向けに調整
4. **新コンポーネント／セクション実装**: サービス紹介、実績一覧、問い合わせCTAなど必要セクションを追加
5. **スタイルとアセット最適化**: 企業ブランド基準に沿って CSS と画像を更新

## 4. 補足タスク
- **ドキュメント更新**: `README.md` に目的とセットアップ手順を追記
- **品質確認**: `npm run lint` と `npm run build` による連続チェック、主要ページのビジュアル確認
